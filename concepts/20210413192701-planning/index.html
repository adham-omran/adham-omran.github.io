Abstract

My plans come in a hierarchical form, starting at any point in the chain can lead you to the other levels of planning. The entire form is as follows:

Planning Hierarchy

  • Vision: Highest level, least defined, most abstract, it’s where you want to find yourself, Example: Become a web developer.

  • Roadmap: A set of projects that when done, accomplish part of a vision. Example: Learn Front-end web development to become a web developer.

    I recommend having milestones when certain parts of the roadmap get done, to have a sense of achievement and to better estimate the time the roadmap and its projects will take.

    Example: A milestone is accomplished when the learner finishes learning HTML during the front-end web development roadmap to become a web developer.

  • Project: A set of tasks and key results to be worked towards, this level is the most defined and where immediate action takes place.

    • Taking Action

      Once you define what your tasks are, decide on what to do next, and then during a weekly review (start doing weekly reviews if you haven’t already), set a time during the week in which you’ll focus on accomplishing this specific task.

    Example: Learn multimedia embedding for HTML to advance the Learn HTML project to advance front-end web developer roadmap to become a web developer.

  • Visions form Roadmaps which contain Projects that contain Tasks and Key Results.

    • Key Results: Desired outcomes from finishing the project.
    • Tasks: TODO items that get worked on.

Questions

I hope these questions can get you started on a plan of interest to yourself.

  • Questions to answer:
    • What am I working towards now?
    • What am I working on for the next X months/Y quarters?
    • Am I doing this as a career or as a hobby?
    • How am I publishing this to the world?
