Abstract

  • Below is the detailed documentation of how I study, by that I mean the process of taking a source of information, converting it into notes, then state how to process these notes, mainly for spaced repition and Zettelkasten literature notes

Purpose of Material

  • For a piece of material to be worth studying, processing and memorizing, it must provide value to the learner.
  • Examples of value:
    • Introduces a new framework or workflow.
    • Improves an existing framework or workflow.
    • Advances my knowledge in a field of interest.
    • Enhances the way I use my existing tools and resources .

Processing Material

Framework

  • When a reading time block starts, check the reading queue to know what to start or continue processing.

  • Keep in mind that the process is a bit different depending on source but the core of (Source -> Dailies -> Zettelkasten / SRS) remains the same.

  • What “Source” gets processed is determined by reading queue (What to read), time categories and deep work blocks (When to read it?).

  • The output of writing notes has two paths.

    1. Exported as .html files then used as input for SuperMemo .
    2. Put inside index notes, then queued and formulated into literature notes.

      • Why index notes?

        With the large number of sources out there, it makes it more easy to work with index notes of “source” each containing literature notes on it.

      • Why use dailies and note take notes directly to the index note?

        Firstly, It allows for a distraction free note-taking environment, as you’re not looking at what you’ve written before and distract yourself by formulating this and that.

        Secondly, It allows for more efficient exports, as you only need to export one branch (org-mode specific).

  • For working on original non-music material, check the writing queue .

Practical Process

  • Create or go to an index note for the material that you’re processing.
  • Open your dailies note for the day.
  • Start a pomodora.
  • Read, Notate, Repeat until pomodoras end.
  • After you finish a chapter or complete a section, export to html then into import into , then move the notes to the index note.

Things to keep in mind while reading

  • Understanding a concept doesn’t mean that you know how to apply it.
  • Re-write the concept in your own words.
  • Try to explain the concept to someone else.
  • Find areas in which you can apply the concept that you’re studying.
  • What’s interesting about this to me?
  • How is this relevant to what I already know?
  • How might this connect to concepts I know about?
